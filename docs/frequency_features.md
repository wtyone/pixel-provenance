# 频域特征说明

频域分析用于从图片纹理、能量分布、相位结构和像素统计中提取辅助线索。它适合回答“这张图是否有异常的频率或像素模式”，不适合单独回答“这张图是不是 AI 生成”。

当前实现会先在主线程解码图片，再中心裁切并缩放到 256-1024 的 2 的幂正方形，随后在 Web Worker 中提取 65 个特征并进行启发式评分。

## 特征类别

| 类别 | 代表特征 | 作用 |
| --- | --- | --- |
| 频谱能量 | 低/中/高频占比、频谱斜率、频谱平坦度、频谱熵 | 观察能量是否接近自然照片常见的 1/f 衰减，或是否过于平坦 |
| 子频带分布 | 0-5%、5-10%、10-20% 等径向频带比例 | 定位异常能量集中在哪个频率区间 |
| 角向结构 | 主方向、方向强度、角向能量方差 | 观察纹理是否具有明显方向性，或是否过度均匀 |
| 相位特征 | RGB 相位一致性、跨通道相位相关 | 捕捉可能由水印、重采样或特殊处理引入的相位结构 |
| LSB 与位平面 | LSB 偏置、二阶 LSB 偏置、低位相关、卡方统计 | 观察低位比特是否偏离常见随机性 |
| 像素统计 | RGB 均值、标准差、偏度、峰度、通道相关 | 判断颜色分布是否异常平滑或相关性异常 |
| 空间相关 | 横向、纵向、对角相邻像素相关，低位断裂比例 | 观察过度平滑、强噪声或二次处理痕迹 |
| 小波特征 | HH/LH/HL 子带能量、小波熵、小波峰度 | 多尺度观察高频细节与纹理复杂度 |
| DCT 块统计 | 8x8 DCT 系数、零系数比例、块间亮度方差 | 观察 JPEG 块结构、压缩残留和局部均匀性 |

## 评分逻辑

评分函数位于 `src/frequency/score.js`。它不是训练模型，而是把若干规则转成加权投票：

- 频谱斜率过平、频谱平坦度高：提示能量分布不像普通自然照片。
- 方向强度低：提示纹理方向性较弱，但也可能来自模糊、降噪或插画。
- 相位一致性或跨通道相位相关异常：提示可能存在水印、重采样或其他处理结构。
- LSB 偏置：提示低位比特分布异常，可能来自处理、隐写、水印或压缩链路。
- 像素峰度过低、通道相关异常：提示颜色统计偏离常见照片，但不是来源证明。
- 相邻像素相关极高：提示过度平滑、降噪或生成/编辑处理。
- 小波高频能量过低、DCT 块间方差低：提示细节能量或块结构偏弱。

评分结果按总分分为：

| 总分 | 文案 | 含义 |
| --- | --- | --- |
| `>= 6` | 频域特征高度异常 | 多条弱规则同时触发，需要人工复核 |
| `3-5` | 存在生成或处理相关频域特征 | 有若干异常，但不能直接判定来源 |
| `1-2` | 轻微信号 | 仅提示进一步查看 |
| `<= -1` | 更接近自然照片纹理 | 当前规则更接近自然纹理，但不能证明真实拍摄 |
| `0` | 特征模糊，无法判定 | 没有足够启发式信号 |

## 为什么不把频域作为强证据

同一张图片经过以下操作后，频域特征都可能明显变化：

- JPEG / WebP 重编码。
- 社交平台压缩。
- 截图、缩放、裁切。
- 降噪、锐化、滤镜。
- HDR、夜景、多帧合成。
- 局部修图或拼接。

因此，频域结果应该被视为“排查线索”。更可靠的证据通常来自可验证的来源凭证、明确的元数据声明、原始文件链路和上下文材料。

## 已知限制

- 当前没有训练集校准，也没有输出置信概率。
- C2PA 只做结构嗅探，不做签名验证。
- 现代生成模型、手机计算摄影和社交平台重编码之间可能高度相似。
- 低分不代表图片真实，高分也不代表图片一定由 AI 生成。
- 频域分析前会中心裁切为正方形，极端构图可能丢失边缘信息。
